# === Commit Message Template ===
# <type>: <short summary>
# 
# <longer description, if needed>
# 
# TYPES:
#   feat     = new feature
#   fix      = bug fix
#   refactor = code restructure without feature/bug changes
#   style    = formatting, naming, no logic changes
#   docs     = documentation only
#   chore    = maintenance, build, config
#   test     = add or update tests
#
# Example:
# feat: add offline gear caching
#
# Allows viewing gear data without network.
